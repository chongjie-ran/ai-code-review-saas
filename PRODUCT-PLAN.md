## 一、用户场景分析

### 场景1: 个人开发者（高频刚需）
**用户画像**: 使用Cursor/GitHub Copilot的独立开发者
**痛点**: AI生成的代码看着能用，但隐藏逻辑错误，上线后Bug一堆
**需求**: 写完AI代码后快速验证，不想切换IDE
**付费意愿**: $0-29/月

### 场景2: 小团队Tech Lead（效率驱动）
**用户画像**: 5-15人创业团队Tech Lead
**痛点**: 团队都在用AI编程，但代码质量参差不齐，没有统一标准
**需求**: CI/CD自动扫描，PR前发现问题，不用人工Code Review那么多轮
**付费意愿**: $99-299/月

### 场景3: 企业安全合规（强需求）
**用户画像**: 金融/医疗/政府行业IT负责人
**痛点**: AI代码可能有安全漏洞，需要满足审计要求
**需求**: 合规报告、权限审计、SOC2等认证支持
**付费意愿**: $299+/月

### 场景4: 外包/离岸团队管理者（质量管控）
**用户画像**: 管理外包团队的PM
**痛点**: 外包团队用AI生成代码，质量不可控
**需求**: 交付前自动质检，出问题有证据链
**付费意愿**: $99-199/月

## 三、MVP功能详细规格（Phase 1）

### API设计

```
POST /api/v1/analyze
- 输入: code + language
- 输出: issues[] + summary + score
- 支持语言: Python, JavaScript, TypeScript

POST /api/v1/analyze/file
- 输入: file_path + language
- 输出: issues[] + summary + score
- 支持CI/CD集成

GET /api/v1/report/{session_id}
- 输出: 完整分析报告

GET /api/v1/health
- 健康检查
```

### 规则库优先级

**P0 必选规则（Phase 1必须覆盖）:**
- AI-LG-001: 返回None可能（空指针风险）
- AI-LG-002: 未处理的异常
- AI-LG-003: 数组索引越界风险
- AI-SEC-001: 硬编码凭证/密钥
- AI-SEC-002: SQL注入风险
- AI-SEC-003: 命令注入风险
- AI-SEC-004: 不安全的反序列化
- AI-DEBT-001: 重复代码块检测
- AI-DEBT-002: 过时API使用警告
- AI-DEBT-003: 过长函数警告

### 前端MVP

- 单页应用，输入代码，输出分析结果
- 支持代码高亮
- 问题列表按严重级别分类
- 一键复制修复建议
