### 二、技术债务清单

#### 高优先级 (影响安全性)

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| TD-01 | 密码用SHA256无盐 | auth.py | hash_password() 使用裸SHA256，应用bcrypt或argon2 |
| TD-02 | CORS allow_origins=["*"] | main.py | 生产环境风险极大，应配置白名单 |
| TD-03 | JWT_SECRET默认值 | auth.py | 未设置时用随机生成，重启后token失效 |

#### 中优先级

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| TD-04 | CI workflow是空壳 | .github/workflows/ci.yml | 实际什么都没做 |
| TD-05 | .db文件进镜像 | backend/ | 数据库文件可能被Docker层持久化 |
| TD-06 | 无数据库迁移 | - | SQLite schema直接用CREATE TABLE IF NOT EXISTS |
| TD-07 | 测试覆盖率极低 | tests/test_analyzers.py | 仅4个测试函数 |
| TD-08 | 规则引擎无AST | rule_engine.py | 全是正则单行匹配，无法检测跨行漏洞 |
| TD-09 | 日志无结构化 | main.py | 仅basicConfig，缺request_id、trace_id |
